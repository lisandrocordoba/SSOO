## EJ 12: Format String Vulnerability

```c
void saludo(void) {
    char nombre[80];
    printf("Ingrese su nombre: ");
    gets(nombre);
    printf(nombre);
    exit(0);
}
```

Sigue habiendo vulnerabilidad de format string. 
Como $saludo()$ no retorna, no serviria sobreescribir el return address. Sin embargo, un atacante aun puede ver punteros de memoria con `%p` o romper el programa con `%s%s%s%s%s%s%s%s`.
Ademas, Ahora tambien hay vulnerabilidad de buffer overflow por el uso de `gets()`.
