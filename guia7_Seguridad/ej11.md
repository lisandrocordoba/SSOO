## EJ 11: Format String Vulnerability
Supongamos que el usuario controla stdin
```c
#define MAX_BUF 4096
    void saludo(void) {
    char nombre[MAX_BUF];
    printf("Ingrese su nombre: ");
    fgets(nombre, MAX_BUF, stdin);
    printf(nombre);
}
```

### a)
El problema de seguridad es no tener format string en el segundo printf. Es decir, no tener `printf("%s", nombre)`.
El usuario puede ingresar `%s%s%s%s%s%s%s%s` y el programa intentará interpretar cada `%s` como un puntero a string, empezando desde la ubicacion del buffer. Leerá de la pila 8 veces y probablemente llegue a una direccion invalida, rompiendo el programa.  
El usuario tambien podria ingresar `%p` forzando al programa a imprimir direcciones de memoria. Claramente este no es un comportamiento esperado.  
Con `%n` el usuario tambien podria escribir en memoria, lo cual es aun peor.
